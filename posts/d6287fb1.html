open-source，阅读并分析代码得出结果

1.描述

　　题干：菜鸡学逆向学的头皮发麻，终于它拿到了一段源代码。【注解：阅读源代码并尝试运行，得出最终的值】
　　场景：暂无
　　来源：HackYou CTF
　　附件：文件名为“8b6405c25fe447fa804c6833a0d72808.c”，实际上是一个文本文件，能够用记事本直接打开，典型的C语言源代码文件。

3.分析一下

3.1 入口函数

　　C语言程序中的main函数是整个程序的入口函数，程序最先进入此函数执行，可以带参数，也可以不带参数。代码中 main 函数带有两个参数，int argc 和 char *argv[] 。 argc 表示参数个数，argv 表示每个参数字符串的首字符地址，因为有多个参数，所以 argv 是一种数组的形式，也可以写成 char **argv 。例如某个C语言程序文件名为 hack.exe ，若要带参数运行可以在Windows cmd命令行中输入如“ hack.exe -a -b -c -d ” 的形式。常见的ping命令也是如此，例如“ ping 127.0.0.1 -t”， 127.0.0.1 和 -t 均为参数。 127.0.0.1 表示ping IP地址为 127.0.0.1 的机器，-t 表示Ping指定的主机直到用户主动停止。

3.2 参数分析

　　从代码当中的if(argc!=4)能够知晓该程序需要4个参数，否则会打印字符串“what？”然后退出程序，返回错误代码1。所以以下分析这四个参数分别是什么。注意数组下标从0开始。
　　参数1： argv[0]，代码当中没有提到，可以先忽略。
　　参数2： argv[1]， if (first != 0xcafe)，代码使用了atoi函数，将字符串转换为int类型的数。函数会扫描参数 nptr字符串，会跳过前面的空白字符（例如空格，tab缩进）等。如果 nptr不能转换成 int 或者 nptr为空字符串，那么将返回 0。很明显参数2在转换后必须等于0xCAFE（十六进制），否则会输出“you are wrong, sorry.”，然后退出程序，返回错误代码2。分析得知参数2为51966（十进制）。
　　参数3： argv[2]，if (second % 5 == 3 || second % 17 != 8)，判断参数3“被5除余数3或者被17除余数不等于18的”。那么反过来应该是“被5除余数不为3且被17除余数为8”，其实同时满足这两个条件的可以是25、42、59……
　　参数4： argv[3]，if (strcmp("h4cky0u", argv[3]))。strcmp函数是string compare(字符串比较)的缩写，用于比较两个字符串并根据比较结果返回整数。基本形式为strcmp(str1,str2)，若str1=str2，则返回零；若str1<str2，则返回负数；若str1>str2，则返回正数。那么0以外的值，无论是正数还是负数在if判断中都被视为真值，所以参数4必须为“h4cky0u”，strcmp返回值为0。

3.3 计算最终的hash变量

　　分析了每个参数之后就可以算出变量hash的值了，即题目的解答。unsigned int hash = first * 31337 + (second % 17) * 11 + strlen(argv[3]) - 1615810207; 要根据这个表达式计算出最终结果。
　　汇总一下：
　　first = argv[1] = 51966
　　(second % 17) * 11 = 8 * 11 = 88
　　strlen(argv[3]) = 7 【注：strlen所作的是一个计数器的工作，它从内存的某个位置（可以是字符串开头，中间某个位置，甚至是某个不确定的内存区域）开始扫描，直到碰到第一个字符串结束符’\0’为止，然后返回计数器值(长度不包含’\0’)。】
　　所以hash = 51966 * 31337 + 88 + 7 - 1615810207 = 12648430

3.4 结果需要转换为十六进制

　　这一步特别重要！因为后面还有一句代码printf("%x\n", hash); 还要注意转换得到十六进制字母全部为小写！因为是“%x”中的x是小写。算是一个小小的陷阱吧。把以上的12648430（十进制）转换成十六进制得到c0ffee。结果也挺巧的，很像咖啡的英语“COFFEE”。

4.总结

　　C语言程序入口函数参数；atoi；strlen；strcmp；转义字符以及大小写；数值进制转换
